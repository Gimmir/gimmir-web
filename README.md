# Gimmir — corporate site

Next.js 16 + Sanity foundation for the Gimmir studio site.

> **Current state:** two designed pages are live — **Home (`/`)** and
> **The Review (`/the-review`)** — ported pixel-faithfully from the mockups in
> `Gimmir Website/`. The supporting **infrastructure** (Sanity CMS, embedded
> Studio, schemas, Draft Mode / Visual Editing, SEO routes, lead backend,
> revalidate webhook) is in place. The two pages are currently **static** (copy
> baked from the mockups); wiring them to Sanity is a later step.

## Design pages

- **Home** and **The Review** are rendered from the design mockups: their
  static markup (`src/components/design/{home,review}-html.ts`, generated by
  `scripts/extract-design.mjs` from `Gimmir Website/*.dc.html`) is injected,
  and the mockups' interactions are reimplemented in
  `src/components/design/runtime.tsx` (scroll reveals, stat counters, sticky
  nav, scroll progress, hover illustrations).
- Fonts: **Archivo** + **Newsreader** self-hosted via `next/font`. Palette:
  paper `#F6F4EE`, ink `#15140E`, lime `#C9F23D`.
- Re-generate the page markup after editing a mockup: `node scripts/extract-design.mjs`.

## What's in place

- **Embedded Sanity Studio** at `/studio`.
- **Content model** (`src/sanity/schemaTypes/`):
  - Singletons: `siteSettings`, `navigation`, `homePage` (page builder).
  - Documents: `caseStudy`, `founder`, `testimonial`, `redirect`.
  - Objects: `seo`, `blockContent`, `socialLink`, `metric`, `cta`, `stat`,
    `processStep`, and page-builder sections (`heroSection`,
    `caseStudiesSection`, `processSection`, `foundersSection`,
    `reviewCtaSection`, `logosSection`, `testimonialSection`).
- **Live content + drafts:** `defineLive` / `sanityFetch` (`src/sanity/lib/live.ts`),
  Draft Mode routes (`/api/draft-mode/{enable,disable}`) and `<VisualEditing />`
  wired in the root layout for click-to-edit previews.
- **GROQ + TypeGen:** typed queries in `src/sanity/lib/queries.ts`, generated
  types in `src/sanity/types.ts` (`npm run typegen`).
- **SEO routes:** `app/sitemap.ts` (pulls published URLs from Sanity, excludes
  `noIndex`), `app/robots.ts`, `app/manifest.ts`, plus managed redirects from
  the `redirect` type via `next.config.ts`.
- **Leads backend:** server action `src/app/actions/lead.ts` (zod validation,
  honeypot, in-memory rate-limit) → Attio (`src/lib/attio.ts`, upsert Person +
  Note). No-ops safely until `ATTIO_API_KEY` is set.
- **Revalidate webhook:** `POST /api/revalidate` (signature-checked,
  tag-based revalidation).
- **Pages:** `/` (Home) and `/the-review` (The Review), plus `/studio`.

## Environment variables

Copy `.env.example` → `.env.local` and fill in. `.env.local` is already
populated for local dev.

| Var | Purpose |
|---|---|
| `NEXT_PUBLIC_SANITY_PROJECT_ID` | Sanity project id (`c4ts5q7s`) |
| `NEXT_PUBLIC_SANITY_DATASET` | `production` |
| `NEXT_PUBLIC_SANITY_API_VERSION` | `2026-02-01` |
| `SANITY_API_READ_TOKEN` | Viewer token — Draft Mode / Visual Editing |
| `SANITY_API_WRITE_TOKEN` | Editor token — only for `npm run seed` |
| `SANITY_REVALIDATE_SECRET` | Shared secret for the `/api/revalidate` webhook |
| `ATTIO_API_KEY` | Attio API key for the lead form (optional until launch) |
| `NEXT_PUBLIC_SITE_URL` | Site origin, e.g. `https://gimmir.com` |

## Commands

```bash
npm run dev        # dev server (http://localhost:3000, Studio at /studio)
npm run build      # production build
npm run start      # serve the production build
npm run lint       # eslint
npm run typecheck  # tsc --noEmit
npm run typegen    # extract schema + regenerate src/sanity/types.ts
npm run seed       # seed sample content into Sanity (uses the write token)
```

## Notes

- **Document IDs must not contain dots.** The Live Content API's perspective
  layer treats a dot as a draft/version separator, so dotted ids (`caseStudy.un1t`)
  return `null` through `sanityFetch`. Use plain ids (`un1t`).
- **Sample content:** `npm run seed` inserts placeholder case studies, founders
  and copy so the Studio isn't empty. Replace or delete it in `/studio`.
- **Attio:** the lead pipeline is implemented but inert until `ATTIO_API_KEY`
  is configured; verify against a test record before launch.
- **Deploy (todo):** Vercel + env vars + Sanity webhook → `/api/revalidate` +
